# План сортировки файлов проекта

## Текущее состояние (кратко)
- Код: `server/`, `client/` организованы нормально.
- Документация: в `docs/` много пересекающихся файлов (часть устарела).
- Временные артефакты лежат в корне: `.diff.txt`, `build_log.txt`, `logs.txt`, `cyrillic_lines.txt`.
- Есть папка `temp_comparison/` с копиями больших файлов.
- Есть дубли в `scripts/` и `scripts/google-apps-script/`.

## Цель сортировки
- Очистить корень проекта.
- Отделить рабочий код от временных/архивных файлов.
- Упростить поддержку и onboarding.

## Безопасный этап (можно делать сразу)
1. Создать служебные папки:
- `tmp/` для логов и временных файлов.
- `archive/` для старых сравнений и снапшотов.
- `docs/archive/` для устаревших инструкций.

2. Перенести артефакты из корня:
- `.diff.txt` -> `tmp/.diff.txt`
- `build_log.txt` -> `tmp/build_log.txt`
- `logs.txt` -> `tmp/logs.txt`
- `cyrillic_lines.txt` -> `tmp/cyrillic_lines.txt`

3. Перенести сравнения:
- `temp_comparison/*` -> `archive/temp_comparison/`

4. Добавить/обновить `.gitignore`:
- `tmp/`
- `archive/`
- `*.log` (если нужно)

## Этап после стабилизации логики (рекомендуется после багфиксов)
1. Документация:
- Оставить актуальные:
  - `docs/BOT_REGISTRATION_REWORK_TASKS.md`
  - `docs/FILE_ORGANIZATION_PLAN.md`
  - `docs/DEPLOYMENT.md` (или единый `docs/DEPLOY.md`)
  - `docs/LOGIN_SYSTEM.md` (после обновления)
- Устаревшие/дубли перенести в `docs/archive/`.

2. Scripts:
- Выбрать один источник правды:
  - либо `scripts/google-apps-script/`,
  - либо `scripts/`.
- Второй вариант архивировать.

3. Утилиты:
- Вынести `check_schema.js` в `scripts/tools/check_schema.js`.
- Обновить README (если будет команда запуска).

## Что не трогаем сейчас
- `server/` и `client/` структуру.
- Миграции/роуты/бот-файлы до завершения багфиксов.
- `uploads/`, `data/` без отдельного решения по хранению.

## Предлагаемый порядок выполнения
1. Завершить багфиксы и стабилизацию регистрации/мультибота.
2. Сделать безопасный этап сортировки.
3. Затем чистка docs/scripts и финальная актуализация README.
