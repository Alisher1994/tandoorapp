## 💡 Ключевые архитектурные решения

### 1. ShowcaseContext вместо Redux
**Причина:** Проще для управления локальным состоянием витрины без излишней сложности.

### 2. JSON в JSONB для layout
**Причина:** Гибкость для расширения структуры блоков в будущем.

### 3. Drag-and-drop HTML5 вместо lib
**Причина:** Нет зависимостей, меньше bundle size, полный контроль.

### 4. Split layout (левая/правая панель) в конструкторе
**Причина:** Идеально для desktop + легко адаптировать на mobile.

### 5. Компоненты блоков вместо динамического рендера
**Причина:** Лучше типизация, проще тестирование, лучше performance.
